Jail Channel Exclusions #

Specify categories/channels you want inmates to see while in jail.

Commands:

/add-channel-exclusion

/remove-channel-exclusion

/show-excluded-channels


Steps
  1. Use the add command and give either a category or channel you would like inmates to see while they're in jail.
  2. Use the show command to see all the categories/channels that inmates can see while they're in jail.
  3. Use the remove command and give either a category or channel you no longer want jail inmates to see while they're in jail.

Enable/Disable JailBots Integrated Ticketing System #

Enabled by default. Disable this if you want to use another ticketing bot to handle inmate disputes.

Commands:

/toggle-integrated-jail-ticketing


Steps
  1. The command itself is pretty self-explanatory - just run it and follow the prompts. If you decide to disable it, the tickets sent to the 'all-tickets' channel will still work. Disabling this just deletes the 'create-a-ticket' channel and the 'Closed Jail Tickets' category if there aren't any channels in it, meaning inmates won't be able to create a ticket with JailBots ticketing system. Use the /add-channel-exclusion command explained above so inmates can still create a ticket with whatever bot you decide to use.

Jail Times Restrictions #

Set restrictions on how long a user can be jailed for.

Commands:

/change-jail-time-restrictions

/show-jail-time-restrictions


Steps
  1. Use the change command to modify the current jailing restrictions. If this is your fist time running this, they will be set to the defaults.
    1. All the arguments for this command are optional, but you have to provide at least one to set a restriction for. Keep in mind all the time restrictions you set are combined to a total. For example, if I tell it to set the max_months=1 and max_weeks=4, it will combine these to mean that a user can't be jailed for longer than 1 month and 4 weeks. Once this is set, if someone tries to jail a user for longer than this, it will be denied. This doesn't include the 'Forever' (infinite) jail sentence option which can be enabled/disabled separately.
    2. After you give it your restrictions, it will ask if you want to set the restrictions for the regular jail (/jail), jail vote (/start-jail-vote), or both.
  2. Use the show command to see the currently set jail time restrictions.

Discord AutoMod Integration #

Jail users when a Discord AutoMod rule takes action on them.

Commands:

/add-automod-rule-jail

/remove-automod-rule-jail

/show-all-automod-rule-jails


Steps
  1. Use the add command and give the ID of the Discord AutoMod rule you've setup along with the time the user should be jailed for when it gets triggered. Keep in mind the times given will be combined into one sentence length. For example, if I enter weeks=2 and days=3, the user that triggers this rule will be jailed for 2 weeks AND 3 days.
    1. To get the AutoMod rule ID, first enable developer mode in your Discord client: User Settings>Advanced>Developer Mode.
    2. Next, click the 3 dots to the right of the AutoMod rule and select `Copy AutoMod Rule ID`. If there's no option to copy the ID, it means you haven't setup that rule yet.
  2. Use the show command to see all the AutoMod rules that will jail users that trigger them.
  3. Use the remove command and give it the AutoMod rule ID you wish to no longer jail users on when it's triggered.
