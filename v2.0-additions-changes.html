JailBot v2.0 Feature Additions and Changes

Coming soon... (these changes should be live sometime on 2/24/24)


Feature Additions

  • Custom Jail Times
    • Allows jailers to specify the exact number of months, weeks, days, hours, and minutes a user should stay in jail for both the /jail and /jail/start-jail-vote commands.
  • Integrated Ticketing System
    • JailBot now has an integrated ticketing system that shows the inmate a "create-a-ticket" channel where they can open a ticket with the JailMods. It's now built into JailBot, which means people no longer need to use another bot such as TicketingTool (although they absolutely still can, it's a great ticketing bot).
  • Jail Times Restrictions
    • Server admins can now set restrictions on how long a user can be jailed for. Using the /change-jail-time-restrictions and /show-jail-time-restrictions commands, a maximum sentence length can be set and includes being able to enable or disable the "forever" (infinite) option. You can set the restrictions independently for both the /jail and /start-jail-vote commands.
  • Discord AutoMod Integration
    • Server admins can now jail users when their Discord AutoMod rule takes action on a user. An admin can use the commands /add-automod-rule-jail, /remove-automod-rule-jail, and /show-all-automod-rule-jails to specify what AutoMod rules to jail a user on when it is triggered and how long that user should be jailed for.

Changes

  • Removed dispute sentence feature.
  • Refreshed/improved the look of jail notifications and tickets.
  • The "staff-commands" channel can now be deleted, and it will stay deleted.
